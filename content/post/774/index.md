## 说明

昨天服务器上传的时候出错，导致产生了乱码文件，使用`rm`命令无法删除，记录一下。

### 步骤

分两种情况；1，0K文件或空文件夹；2，大于0K文件或非空文件夹

#### 0K文件或空文件夹

执行`ls`查看到乱码文件如下：

![乱码文件](https://cdn.rumosky.com/usr/uploads/2024/05/173099617.png)

这几个文件都是0KB，首先执行：`ls -i`查看文件ID：

![查看ID](https://cdn.rumosky.com/usr/uploads/2024/05/3751483612.png)

执行下面的命令删除文件：

```bash
find -inum 533176 -delete # 数字为文件ID
```

#### 大于0K文件或非空文件夹

首先，也是查文件ID，执行`ls -i`，然后执行下面的命令删除文件：

```bash
find -inum 533176 -exec rm -rf {} \
```

会提示找不到此文件或文件夹，但其实已经删除了

## 最后

一般文件乱码是上传错误导致的，也有的是编码问题，确定清楚再删除。
